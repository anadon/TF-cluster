Affymetrix® Fusion SDK - Release 1.1.2 (June 2011)

INTRODUCTION

The Fusion SDK provides ANSI C++ source code for parsing data files generated and managed by the Affymetrix GeneChip Command Console software and the older GCOS 1.x software. Using the Fusion SDK provides backwards and forward flexibility by providing the ability to parse both types of data files. Newsletters and recorded WebTalks about the Command Console software are available on the Affymetrix DevNet web site .

The SDK was developed using the ANSI C++ language and tested using Microsoft Visual Studio 2008 C++ on 32 and 64 bit Windows 7 systems, G++ 4.4.4 compiler on an Apple OS X 10.6 system and G++ 4.4.4 compiler on 32 and 64 bit Fedora linux systems. The Java code in the Fusion SDK requires JRE 1.6 or above.

The documentation provided and source code in the Fusion SDK contains references to the project code named "Calvin". This code name is used to refer to the Affymetrix GeneChip Command Console software.

The source code in the Fusion SDK is licensed under the GNU Lesser General Public License (LGPL).

RELEASE NOTES

Version 1.1.2 Updates (June 2011)

  • This release of our Fusion SDK provides support for an enhanced version of the CYCHP file format. You can find detailed information about the format in our file formats document included in this package
    • A data group has been added to the CYCHP files generated from the CytoScan HD array to house genotyping data generated by brlmm-p-plus analysis.
    • For CYCHP files generated from CytoScan HD arrays, the ASignal and BSignal values stored in the Algorithm data group have been moved to the new genotyping data group. These metrics will continue to be housed in the Algorithm table for those CYCHP files generated from the 2.7M array
    • The Segments data group is optional and will be included in the CYCHP file if segments are report by an analysis.
    • The Summary data set in the Chromosome data group now includes a column for the MedianSignal value.
  • The C++ and Java versions of the Fusion SDK have been enhanced to seamlessly parse both the CytoScan HD and the 2.7M arrays. Also, functionality to write CYCHP files for the CytoScan HD array has been added to the C++ version of the SDK.
  • The C++ and Java versions of the read_cytov2 sample application have been updated to provide example code for reading the new genotyping table. Also provided is sample code for reading the updated Algorithm table.
  • Upgrades to the C++ SDK
    • The primary platform for the released binary and library files is now 64 bit. All compiled binaries and libraries included in the SDK packages are for 64-bit platforms. The source code included in the Fusion SDK can be used to build 32-bit versions of the binaries and libraries. We continue to provide support for 32-bit platforms in this release
    • The Visual Studio projects for building libraries, sample programs and converter applications have been updated to include configuration settings for 64 bit compilations. The settings for compiling the projects for the 32 bit platform have been retained,
    • The warning level in the Visual Studio projects files generating libraries has been changed from 3 to 0.
    • A new example program to write expression and genotyping results is included.

Version 1.1.1 Updates (March 2011)

  • Support has been added to the C++ and Java SDKs for parsing the new CDF file format version for Axiom™ Genotyping Arrays. This CDF file format allows for multiple versions of CDF Files for a probe array.
    • The document CDF File Format Change describes the new file format version and details the support provided in the Fusion SDK.
    • The API calls added the Fusion SDK can be used to parse any CDF file. The CDF file format version is transparent to the software invoking the API.
  • Upgrades and changes to the C++ SDK
    • The Fusion SDK was upgraded to correctly handle paths longer than 260 characters (MAX_PATH) for the Windows platform. The SDK will convert paths to Microsoft's Long Universal Naming Convention.
    • File operations located in affy/sdk/util/Util.h and affy/sdk/util/AffxFile.h were consolidated into affy/sdk/util/Fs.h.
    • New file open APIs that are Windows Long UNC compliant were introduced.
    • The file APIs for reading CSV files in affy/sdk/util/AffxFile.h have been removed. All CSV data files as well as tab delimited files are now being parsed with affy/sdk/file/TsvFile.
    • The project settings of the Microsoft Visual Studio 2008 project files used to create libraries encapsulating code for parsing and writing Affymetrix files have been changed to enhance performance. Compiled 32 bit versions of the libraries used in our sample applications can be found in the ./libraries_and_tools/libraries folder
    • The name of the Microsoft Visual Studio project used to build the library encapsulating code in the folder affy/sdk/util has been changed to lib-util-devnet.vcproj. The name of the compiled library remains lib-util.lib. A compiled version of the library can be found in the ./libraries_and_tools/libraries folder.
  • Upgrades and changes to the Java SDK
    • The Java source code requires version 1.6 of the JRE.
    • An issue with reading ARR files has been resolved.

COMPILING THE CODE

There are several macros that may need to be defined, using the "-D" compiler switch, to enable compilation and execution on specific operating systems. These are:

Macro Definition Description
IS_BIG_ENDIAN

 Define this value if the machine represents its data in big ENDIAN format. If the machine is a little ENDIAN machine (such as an Intel system) then omit this macro.
_INCLUDE_UNISTD_HEADER_

 Define this value if the UNISTD.H header file is used by the C++ compiler on your machine.
WIN32

Define this value if using the MS VC++ compilers.
_USE_MEM_MAPPING_

 This value is used to indicate if memory mapping should be used by the GCOS 1.x BAR, BPMAP and CDF file parsers. Memory mapping is automatically used for CEL file parsing and by the Command Console file parsers.

3RD PARTY TOOLS

XERCES

The SDK uses the XERCES library for parsing XML files. Source code for this library is not included as part of the Fusion SDK. Current Command Console formatted XML files are:

  • Array Files
  • Templates Files
  • DTT 1.1 XML MAGE Files

The XERCES library is available at: http://xml.apache.org/xerces-c/ . You must download and build this library to use the XML parsing objects of the Fusion SDK. The Fusion SDK was tested with XERCES version 3.0.

XERCES must be installed at the same level as "affy" directory. The folder name must be "xerces". The name and relative location to the affy directory is assumed within each Visual Studio project.

Example:

  • /development/./...
  • /development/xerces

CPPUNIT

Previous versions of the Fusion SDK (version 1 through 1.0.12) included a set of unit test cases to test each class within the SDK. This version of the Fusion SDK does not include unit test cases. The information below is retained for those interested in running the test cases included in earlier versions.

    The unit tests included with earlier versions of the Fusion SDK were developed using the CPPUNIT Framework version 1.10.2. Test case source and Visual Studio Projects were located in the "./calvin_files/*/test" folders. Each Visual Studio Project (located in the test directory - e.g. ./calvin_files/array/test) will compile, build and execute the test cases. The test cases will run giving a "." for each test when it completes. If an "F" is displayed instead of a ".", then that test case failed. When all the test cases were attempted and failed or passed, those that failed will be indicated with the source line and file where the error occurred. The number of passed and failed test cases will be indicated as well. If all test cases pass, then an "OK" is indicated along with the test case count. The test cases may also be performed inside the development environment in debug mode. Stepping through code during debug is also supported (Visual Studio).

    Test data included in the earlier versions of the Fusion SDK could be found in the data folder one level up from the test folders - e.g. ./calvin_files/parsers/data. An additional test data folder is provided for larger files and that is in common to both GCOS 1.x File SDK and Fusion. This test data is located in the "./TestDataFiles" folder

    CPPUNIT must be installed at the same level as "affy" directory. The folder name must be "cppunit". The name and relative location to the affy directory is assumed within each Visual Studio project.

    Example:

    • /development/./...
    • /development/cppunit
