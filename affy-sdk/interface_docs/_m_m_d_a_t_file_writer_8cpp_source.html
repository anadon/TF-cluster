00027 /* General Description of the file writing process
00028  * The MMDATFileWriter extends the DATFileWriter by adding memory-mapped view
00029  * to the pixel DataSet data.
00030  * The DATFileWriter constructor writes the FileHeader, GenericDataHeader, DataGroupHeader
00031  * and DataSetHeader using fstream.  The header positions are computed and written in the proper
00032  * place at construction.
00033  * The Open method opens a memory mapped view on the pixel DataSet data section.  Data can be
00034  * written to the file by changing the data in the memory mapped view.  The view can also 
00035  * be moved over the pixel data to access all the data.  The view size is restricted to 
00036  * 200 MB at a time.
00037  * Closing the file will also cause the pixel data to be written to the file in network byte-order.
00038  * Warning: Using the same DATData object to re-open the file with the MMDATFileWriter would make it
00039  * possible to access the pixel data that is not in host byte-order.  You do not want to do this.
00040  * Once the MMDATFileWriter is closed, the file should be opened with the DATFileReader to access
00041  * the data.  Re-opening a file with the MMDATFileWriter to overwrite an existing file is OK.
00042  */
