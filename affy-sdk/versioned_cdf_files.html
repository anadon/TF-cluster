Affymetrix® CDF Data File Format Change

The format of our CDF files for Axiom™ Genotyping Arrays has been changed to provide support for multple versions of CDF files. CDF files relate the names of the probesets to the features on our arrays. Analyses performed by Affymetrix software provide results for only those probe sets listed in CDF files.

The primary purpose of this change in the data file format is to allow for variance in the probe set content included in our CDF files. New versions of CDF files will be published when the constituency of probe sets change and/or probe sets are added or removed from the analysis. The format of the CDF files for Axiom™ and other GeneChip® arrays that are currently available will not change.

Description of CDF File Content Change

The additions to the file format to allow for versioning of CDF files are noted below. Please note that these additions will also allow us to provide support for those probe arrays where versioning information is included in the name of the array.

Additional details can be found in the Affymetrix Data File Formats documentation for CDF files.

  • A GUID to uniquely identify the CDF file.
  • An md5 value for verifying the integrity of the file.
  • An array of probe array types supported by the CDF file.

Additions to the Fusion SDK

API calls have been added to our Fusion SDK to provide the values of the new fields to calling applications. The Fusion APIs can be used to parse any CDF file. The CDF file format version is transparent to the software invoking the API.

The following summarizes the additions and changes to the Fusion SDK in support of the new CDF file format:

Class Function Name Description
FusionCDFFileHeader GetFormatVersion() Returns the file version of the CDF file
FusionCDFFileHeader GetGUID() Returns the GUID; For CDF file versions 1-3, this call returns NULL.
FusionCDFFileHeader GetIntegrityMd5() Returns the md5 value from the CDF file header
FusionCDFData GetChipType() Returns the name of the probe array without versioning information; In the C++ SDK, the signature was changed to remove "const".
FusionCDFData GetChipTypes() Returns a vector with array names and versions supported by the CDF file (i.e. probeArray.r1, probeArray.r2); In the C++ SDK, the signature was changed to remove "const".
FusionCDFData GetGUID() Returns the GUID; For CDF file versions 1-3, this call returns NULL.
FusionCDFData GetIntegrityMd5() Returns the md5 value; For CDF file versions 1-3, this call returns 0.
